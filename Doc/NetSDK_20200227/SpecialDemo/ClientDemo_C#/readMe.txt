更新后请注意:
注:C#为动态加载,不会用到netsdk.h头文件，
有一些接口有带有默认值，带不带由netsdk.h决定
1、c#中登陆接口中H264_DVR_DEVICEINFO参数有增加成员变量，c#也应相应增加，否则登陆会崩溃，详细见demo修改的部分
2、一些动态加载sdk的方式就不会带有默认值,可能更新Netsdk.dll后调用某些带有默认参数的时候会崩溃(例如按文件名/时间下载录像),那么就需要在c#接口里面加上默认的参数。默认参数可以参考demo修改的部分
3、有些接口demo中没有调用，客户在工程中调用的接口如果更新Netsdk.dll库后该接口产生崩溃，也需要加上默认参数，默认参数可以见bin文件中的netsdk.h文件
